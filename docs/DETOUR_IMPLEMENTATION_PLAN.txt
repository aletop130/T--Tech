ARCHITECTURE NOTES
-------------------
- **Keanucz/Detour inspiration**: The Detour system follows the design of the keanucz/detour project, leveraging a multi‑agent pipeline (Scout → Analyst → Planner → Safety → Ops) orchestrated by LangGraph. The same approach is applied here to orchestrate debris collision analysis and maneuver planning, with agents communicating via a shared graph state.
- **InstancedMesh pattern in Cesium**: Instead of using React‑Three‑Fiber's InstancedMesh, we adopt Cesium's `Primitive` with `GeometryInstance` to render thousands of debris objects efficiently. Each debris is represented by a point geometry with a shared material, using per‑instance attributes for position and color. This enables GPU‑side instancing and reduces draw calls.
- **Performance optimizations**:
  - Batch propagation of TLEs using Skyfield's vectorized API to generate positions for up to 2500 objects within < 50 ms.
  - Debris updates are throttled to a 15 second interval, and only changed objects are re‑uploaded to the GPU.
  - Use of Cesium's `VertexArrayObject` caching and `primitive` `releaseResources` on cleanup.
  - Limiting rendered objects to `maxDisplayObjects` (default 2500) and culling objects outside the camera view.
  - Asynchronous background workers (Celery) for heavy screening and risk calculations, keeping API responses fast.

TASK 2.4: Screening Congiunzioni
--------------------------------
FILE: backend/app/physics/screening.py

FUNZIONI DA IMPLEMENTARE:
1. screen_conjunctions(primary_tle: TLE, catalog: list[TLE], 
                       time_window_hours: float = 72,
                       threshold_km: float = 5.0) -> list[ConjunctionCandidate]
   - Implementa algoritmo di screening geometrico
   - Per ogni oggetto nel catalogo:
     * Propaga entrambi gli oggetti su griglia temporale (step ogni 10 min)
     * Calcola distanza minima approssimata
     * Se < threshold, aggiungi a candidati per refinement
   - Output: Lista candidati con TCA approssimato, miss distance, etc.

2. refine_conjunction(candidate: ConjunctionCandidate, 
                     iterations: int = 10) -> ConjunctionEvent
   - Refinement preciso con iterazioni Newton-Raphson
   - Trova TCA esatto
   - Calcola geometria miss (radial, intrack, crosstrack)
   - Input: candidato dallo screening
   - Output: ConjunctionEvent completo

TASK 2.5: Calcolo Rischio Collisione
------------------------------------
FILE: backend/app/physics/risk.py

FUNZIONI DA IMPLEMENTARE:
1. calculate_collision_probability_chan(primary_cov: Covariance3D,
                                       secondary_cov: Covariance3D,
                                       miss_distance: np.ndarray,
                                       combined_radius: float) -> float
   - Implementa formula Chan 1997 per probabilità collisione
   - Input: matrici covarianza 3x3 entrambi oggetti, vettore miss distance, somma raggi
   - Output: probabilità [0,1]

2. assess_risk_level(collision_prob: float, 
                    miss_distance_km: float,
                    object_sizes: tuple[float, float]) -> RiskLevel
   - Logica decisionale risk level (low/medium/high/critical)
   - Considera: Pc, miss distance, dimensioni oggetti

3. calculate_maximum_conjunction_time(primary_state: StateVector,
                                     secondary_state: StateVector,
                                     threshold_km: float = 10.0) -> float
   - Tempo massimo in cui oggetti sono entro threshold

TASK 2.6: Calcolo Manovre Avoidance
-----------------------------------
FILE: backend/app/physics/maneuver.py

FUNZIONI DA IMPLEMENTARE:
1. calculate_raan_precession_rate(orbit: OrbitElements) -> float
   - Rate precessione nodo ascendente (effetto J2)

2. propose_in_plane_maneuvers(primary: SatelliteState,
                             conjunction: ConjunctionEvent,
                             delta_v_budget: float = 0.5) -> list[ManeuverOption]
   - Calcola manovre in-plane possibili:
     * Aumento/diminuzione velocità (cambia periodo/orbita)
     * Timing ottimale per massimare miss distance
   - Vincoli: delta-v disponibile, finestra temporale, risorse satellite

3. propose_out_of_plane_maneuvers(primary: SatelliteState,
                                 conjunction: ConjunctionEvent,
                                 delta_v_budget: float = 0.5) -> list[ManeuverOption]
   - Calcola manovre out-of-plane (cambio inclinazione/RAAN)
   - Considera costo energetico vs riduzione rischio

4. optimize_maneuver_timing(maneuvers: list[ManeuverOption],
                           conjunction: ConjunctionEvent) -> ManeuverOption
   - Ottimizza timing manovra per massimare miss distance
   - Considera: tempo reazione, errori esecuzione, propagation uncertainty

5. calculate_delta_v_cost(maneuver: ManeuverOption, 
                         satellite: SatelliteState) -> float
   - Calcola costo delta-v manovra specifica

1. @tool
def screen_conjunctions_tool(satellite_id: str, 
                             time_window_hours: int = 72,
                             threshold_km: float = 5.0) -> str:
    """Screen for potential conjunctions in time window."""
    - Chiama physics.screening.screen_conjunctions

def assess_risk_tool(conjunction_event_id: str) -> str:
    """Perform detailed risk assessment on conjunction."""
    - Recupera dati congiunzione
    - Chiama physics.risk.calculate_collision_probability_chan

                          delta_v_budget: float = 0.5) -> str:
    """Propose avoidance maneuver options."""
    - Chiama physics.maneuver.propose_in_plane_maneuvers
    - Chiama physics.maneuver.propose_out_of_plane_maneuvers
    - Ritorna opzioni con trade-off

4. @tool
def validate_maneuver_tool(maneuver_plan_id: str) -> str:
    """Validate maneuver plan for safety."""
    - Verifica vincoli safety
    - Controlla collisioni secondarie
    - Ritorna esito validazione

5. @tool
def execute_maneuver_tool(maneuver_plan_id: str) -> str:
    """Execute approved maneuver plan."""
    - Solo per plan approvati
    - Aggiorna stato satellite

   - Valida ogni piano manovra
   - Verifica vincoli
   - Aggiorna state.safety_review

5. ops_brief_node(state: DetourGraphState, config: RunnableConfig) -> DetourGraphState
   - Genera riepilogo operativo
   - Formatta output leggibile
   - Aggiorna state.ops_brief
   - Imposta state.completed = True

TASK 2.13: Costruzione Grafo LangGraph
--------------------------------------
FILE: backend/app/agents/detour/graph.py

FUNZIONI DA IMPLEMENTARE:

1. def build_detour_graph() -> StateGraph:
   - Crea StateGraph con DetourGraphState
   - Aggiunge nodi: scout, analyst, planner, safety, ops_brief
   - Definisce archi condizionali:
     * scout -> analyst (sempre)
     * analyst -> planner (se risk >= medium)
     * analyst -> ops_brief (se risk < medium)
     * planner -> safety (sempre)
     * safety -> ops_brief (se approvato)
     * safety -> planner (se modifiche richieste, max 3 iterazioni)

2. async def run_detour_pipeline(session_id: str, 
                                satellite_id: str,
                                conjunction_event_id: str,
                                tenant_id: str,
                                state_manager: DetourStateManager) -> DetourGraphState:
   - Compila grafo
   - Inizializza stato
   - Esegue grafo fino a completamento
   - Salva risultati in DB
   - Ritorna stato finale

3. async def stream_detour_pipeline(...) -> AsyncGenerator[AgentEvent, None]:
   - Come run ma yield eventi in tempo reale
   - Usato per SSE streaming
